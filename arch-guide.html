// step-by-step

Install Arch Linux

Follow these steps carefully. Each command is explained so you know what it does — not just how to type it.

01

Download the ISO & Flash to USB

Go to archlinux.org/download and download the latest ISO. Then flash it to a USB drive using a tool like Balena Etcher or Ventoy. Boot from the USB in your BIOS/UEFI settings.

💡

Disable Secure Boot in BIOS before booting. You can re-enable it after install if needed.

02

Connect to the Internet

If using Wi-Fi, use iwctl to connect. Ethernet usually works automatically.

# Start the iwctl interactive shell
iwctl
# Then inside iwctl:
device list
station wlan0 scan
station wlan0 get-networks
station wlan0 connect "YourWiFiName"
# Exit iwctl and test the connection
ping -c 3 archlinux.org
03

Partition Your Disk

Use cfdisk for a simple menu-based partition tool. For most systems, you need at minimum: an EFI partition and a root partition.

# See your drives
lsblk
# Open cfdisk on your target drive (replace sda with yours)
cfdisk /dev/sda
# Recommended layout:
# /dev/sda1  →  512MB  →  EFI System
# /dev/sda2  →  rest   →  Linux filesystem
04

Format & Mount Partitions

Format each partition with the correct filesystem and mount them for installation.

# Format EFI partition
mkfs.fat -F32 /dev/sda1
# Format root partition
mkfs.ext4 /dev/sda2
# Mount root
mount /dev/sda2 /mnt
# Mount EFI
mkdir -p /mnt/boot/efi
mount /dev/sda1 /mnt/boot/efi
05

Install the Base System

Use pacstrap to install the essential packages into the mounted partition.

# Install base packages + text editor + networking
pacstrap /mnt base linux linux-firmware \
         base-devel nano networkmanager grub \
         efibootmgr sudo
⏳

This will take a few minutes depending on your internet speed. Let it finish.

06

Generate Fstab & Enter System

Generate the filesystem table and chroot into your new install to configure it.

# Generate fstab
genfstab -U /mnt >> /mnt/etc/fstab
# Enter the new system
arch-chroot /mnt
07

Configure the System

Set your timezone, locale, hostname, and root password.

# Set timezone (change to your region)
ln -sf /usr/share/zoneinfo/Asia/Bangkok /etc/localtime
hwclock --systohc

# Set locale
echo "en_US.UTF-8 UTF-8" >> /etc/locale.gen
locale-gen
echo "LANG=en_US.UTF-8" > /etc/locale.conf

# Set hostname
echo "myhostname" > /etc/hostname

# Set root password
passwd
08

Create a User

Never use root as your daily driver. Create a regular user with sudo access.

# Create user (replace "yourname")
useradd -m -G wheel yourname
passwd yourname

# Give sudo access to wheel group
EDITOR=nano visudo
# Uncomment: %wheel ALL=(ALL:ALL) ALL
09

Install Bootloader & Enable Services

Install GRUB so your system can boot, then enable networking and exit.

# Install GRUB
grub-install --target=x86_64-efi \
  --efi-directory=/boot/efi \
  --bootloader-id=GRUB
grub-mkconfig -o /boot/grub/grub.cfg

# Enable NetworkManager on boot
systemctl enable NetworkManager

# Exit, unmount, reboot!
exit
umount -R /mnt
reboot
🎉

Remove the USB drive when the system reboots. You should see the GRUB menu. Arch Linux is installed!

Install with archinstall

The archinstall script is included in every Arch ISO. It walks you through setup with a simple menu — no memorising commands needed. You still get a real, pure Arch system at the end.

01

Boot the Arch ISO

Download and flash the Arch ISO exactly like the manual method (see Step 01 above). Boot from the USB. When you see the live terminal prompt, you are ready.

💡

Make sure you are connected to the internet first. For Wi-Fi, use iwctl to connect before running archinstall.

02

Run the archinstall Command

The script is already on the ISO. Just type one command and the guided menu opens.

# Connect to Wi-Fi first if needed (skip if using ethernet)
iwctl
station wlan0 connect "YourWiFiName"
# exit iwctl, then launch the installer
archinstall
03

Choose Your Language & Locale

The first menu asks for your keyboard layout and locale. Use the arrow keys to navigate, Enter to select. For most users: en_US.

Archinstall — Language & Locale
Keyboard layout → us
Locale → en_US.UTF-8
Mirror region → Worldwide
// Use ↑↓ to move, Enter to select, Esc to go back
04

Select Your Drive & Partition Layout

Choose the disk you want to install Arch on. For most users, pick Best-effort default partition layout — it automatically creates the right partitions for you.

Disk configuration
Use a best-effort default partition layout
Manual partitioning
Pre-mounted configuration
// Best-effort creates: EFI + root. Good for most installs.
⚠️

This will erase the selected drive. Make sure you pick the right disk. Run lsblk in another terminal to double check.

05

Set Hostname, Root Password & Create User

Give your machine a name (hostname), set a root password, and create a regular user account. Always create a user — do not use root as your daily account.

System configuration
Hostname → mymachine
Root password → (set one)
User account → yourname [wheel]
sudo → With sudo enabled
// Add your user to the "wheel" group to get sudo access
06

Choose a Profile & Desktop Environment

This is where archinstall shines over manual install. You can pick a full desktop environment and it installs all the packages for you automatically.

Profile
Desktop
→ KDE Plasma ✓ (full desktop, great for beginners)
→ GNOME
→ Hyprland (tiling, wayland, advanced)
→ i3 (minimal tiling WM)
Minimal (no desktop, just terminal)
// KDE or GNOME recommended if you want a regular desktop
07

Pick Audio & Network Manager

Select Pipewire for audio (modern and works great) and NetworkManager so Wi-Fi works after reboot.

Additional packages / services
Audio → Pipewire
Network → NetworkManager
Timezone → Asia/Bangkok
// Pipewire replaces PulseAudio and works better on modern hardware
08

Install & Reboot

Once all options look correct, scroll to Install and press Enter. The script downloads and installs everything. When it finishes, reboot and remove the USB.

# archinstall handles everything. When done, reboot:
reboot
# Remove your USB drive when the screen goes black
# Your new Arch system will boot with the desktop ready
🎉

Done! Log in with the user you created. Run sudo pacman -Syu to update everything right away.

Install yay — AUR Helper

The official Arch repos have thousands of packages, but the AUR (Arch User Repository) has tens of thousands more — including Google Chrome, Brave, Spotify, and almost anything else you need. yay is the most popular tool for installing AUR packages easily.

01

Install Required Dependencies

yay needs git and the base-devel group to build packages. Install them first with pacman.

# Make sure these are installed
sudo pacman -S --needed git base-devel
02

Clone & Build yay

Download the yay source from the AUR and build it. This only needs to be done once. After this, yay updates itself.

# Clone yay from the AUR
git clone https://aur.archlinux.org/yay.git
cd yay

# Build and install yay
makepkg -si

# Type Y and press Enter when asked to install
⚠️

Never run makepkg as root or with sudo. It must be run as a regular user.

03

Verify yay is Working

After install, test that yay works and check its version.

# Check yay version
yay --version

# Search for any package (test it out)
yay -Ss brave
04

Basic yay Commands

yay works just like pacman but also searches the AUR. Here are the commands you will use most.

# Install a package (searches both pacman + AUR)
yay -S package-name

# Search for a package
yay -Ss keyword

# Update ALL packages including AUR
yay -Syu

# Remove a package
yay -R package-name

# Remove a package AND its unused dependencies
yay -Rns package-name
// web browsers

Install Browsers with yay

Google Chrome and Brave are not in the official Arch repos — they live in the AUR. With yay installed, adding them is just one command each.

🌐

Google Chrome

The most widely used browser. Fast, compatible with everything, and has the largest extension library. Uses your Google account to sync bookmarks and passwords.

# Install Google Chrome (stable)
yay -S google-chrome

# Or install unstable/beta builds:
# yay -S google-chrome-beta
# yay -S google-chrome-dev
🦁

Brave Browser

Privacy-focused browser based on Chromium. Blocks ads and trackers by default, no setup required. Fast, and compatible with Chrome extensions.

# Install Brave (stable)
yay -S brave-bin

# brave-bin = pre-built binary (faster to install)
# brave = compiled from source (slower)
💡

During AUR installs, yay will show you the PKGBUILD and ask to review it. Press N to skip the review and Y to confirm installation. Always say yes to proceed.

Want more browsers? Here are a few others popular on Arch:

# Firefox (in official repos, no yay needed)
sudo pacman -S firefox

# Vivaldi (AUR)
yay -S vivaldi

# Floorp - privacy Firefox fork (AUR)
yay -S floorp-bin

How to Update Arch Linux

Arch is a rolling release — there is no version upgrade process. You just update regularly and your system stays current. Here is everything you need to know.

🔄 Update Everything (Recommended)

Updates all official packages AND AUR packages in one command. Use this as your regular update routine.

# Full system update with yay
yay -Syu

# Or just pacman (no AUR)
sudo pacman -Syu

🧹 Clean Package Cache

Old package versions pile up on disk. Clean the cache regularly to free space. Keep at least 2 old versions in case you need to downgrade.

# Remove all but last 3 versions
sudo paccache -r

# Remove ALL cached packages
sudo pacman -Scc

🗑️ Remove Orphan Packages

Orphans are packages that were installed as dependencies but are no longer needed. Safe to remove.

# List orphan packages
pacman -Qdt

# Remove all orphans
sudo pacman -Rns $(pacman -Qdtq)

🔑 Refresh Keyring

If you get GPG keyring errors during updates, refresh the Arch keyring. This fixes most signature errors.

# Refresh Arch Linux keyring
sudo pacman -S archlinux-keyring
sudo pacman-key --refresh-keys
📅

Update at least once a week. Arch moves fast — waiting months between updates can cause dependency conflicts. If something breaks after an update, check the Arch Linux News page first.

🛟 How to Downgrade a Package

If an update breaks something, you can go back to an older version.

# Install the downgrade tool
yay -S downgrade

# Then run it for any package
sudo downgrade package-name

# Example: downgrade firefox
sudo downgrade firefox

Install Hyprland

Hyprland is a dynamic tiling window manager for Wayland. It is highly customisable, smooth, and looks stunning with the right dotfiles. This guide gets you from a bare Arch install to a working Hyprland desktop.

01

Install Hyprland & Core Dependencies

Install Hyprland and the essential packages needed to get a working desktop: a terminal, status bar, application launcher, notification daemon, and authentication agent.

hyprland kitty waybar wofi hyprpaper dunst polkit-kde-agent xdg-desktop-portal-hyprland 
# Install Hyprland and essential packages
sudo pacman -S hyprland kitty waybar wofi \
               hyprpaper dunst \
               polkit-kde-agent \
               xdg-desktop-portal-hyprland \
               qt5-wayland qt6-wayland
💡

kitty = terminal · waybar = status bar · wofi = app launcher · hyprpaper = wallpaper · dunst = notifications

02

Install Audio & Network Stack

Hyprland does not pull in audio or networking automatically. Install Pipewire for sound and NetworkManager for internet.

# Audio (Pipewire)
sudo pacman -S pipewire pipewire-alsa pipewire-pulse \
               pipewire-jack wireplumber

# Network
sudo pacman -S networkmanager nm-connection-editor
sudo systemctl enable --now NetworkManager

# Enable Pipewire on login
systemctl --user enable --now pipewire pipewire-pulse wireplumber
03

Install a Display Manager (Login Screen)

A display manager shows your login screen and launches Hyprland. SDDM is the most commonly used with Hyprland.

# Install SDDM
sudo pacman -S sddm

# Enable SDDM to start on boot
sudo systemctl enable sddm

# Optional: install a nice SDDM theme from AUR
yay -S sddm-theme-astronaut
04

Create the Hyprland Config

Hyprland's config file lives at ~/.config/hypr/hyprland.conf. On first launch it creates a default one. Here is the minimal config to get started:

# Create config folder
mkdir -p ~/.config/hypr

# Create a basic hyprland.conf
nano ~/.config/hypr/hyprland.conf
# Paste this into hyprland.conf as a starting point

monitor=,preferred,auto,1

exec-once = hyprpaper
exec-once = waybar
exec-once = dunst
exec-once = /usr/lib/polkit-kde-authentication-agent-1

$terminal = kitty
$menu     = wofi --show drun

general {
    gaps_in  = 5
    gaps_out = 10
    border_size = 2
    col.active_border = rgba(ff6ac1ff) rgba(57c7ffff) 45deg
    col.inactive_border = rgba(1e2530ff)
    layout = dwindle
}

decoration {
    rounding = 10
    blur { enabled = true; size = 6; passes = 2 }
    drop_shadow = true
    shadow_range = 12
    shadow_color = rgba(0,0,0,0.5)
}

animations {
    enabled = true
    bezier = myBezier, 0.05, 0.9, 0.1, 1.05
    animation = windows, 1, 5, myBezier
    animation = fade, 1, 5, default
    animation = workspaces, 1, 4, default
}

# Key bindings
$mod = SUPER

bind = $mod, Return,  exec,      $terminal
bind = $mod, D,       exec,      $menu
bind = $mod, Q,       killactive
bind = $mod, F,       fullscreen
bind = $mod SHIFT, E, exit

# Move focus with SUPER + arrow keys
bind = $mod, left,  movefocus, l
bind = $mod, right, movefocus, r
bind = $mod, up,    movefocus, u
bind = $mod, down,  movefocus, d

# Switch workspaces with SUPER + [1-9]
bind = $mod, 1, workspace, 1
bind = $mod, 2, workspace, 2
bind = $mod, 3, workspace, 3
bind = $mod, 4, workspace, 4
bind = $mod, 5, workspace, 5

# Move window to workspace with SUPER + SHIFT + [1-9]
bind = $mod SHIFT, 1, movetoworkspace, 1
bind = $mod SHIFT, 2, movetoworkspace, 2
bind = $mod SHIFT, 3, movetoworkspace, 3

# Mouse: drag windows with SUPER + LMB/RMB
bindm = $mod, mouse:272, movewindow
bindm = $mod, mouse:273, resizewindow
05

Set a Wallpaper with hyprpaper

Create the hyprpaper config file to set your wallpaper. Replace the path with the actual path to your image file.

# Create hyprpaper config
nano ~/.config/hypr/hyprpaper.conf
# Paste this into hyprpaper.conf
# Replace the path with your wallpaper file

preload  = ~/Pictures/wallpaper.jpg
wallpaper = ,~/Pictures/wallpaper.jpg
🖼️

Place a wallpaper image in ~/Pictures/ first. Supported formats: JPG, PNG, WebP.

06

Install Useful Extras

These are not required but strongly recommended for a comfortable Hyprland desktop experience.

# Screenshot tool
sudo pacman -S grim slurp

# Clipboard manager
sudo pacman -S wl-clipboard cliphist

# File manager (GUI)
sudo pacman -S thunar

# Screen lock
yay -S hyprlock

# Idle daemon (auto-lock screen)
yay -S hypridle

# Better app launcher (alternative to wofi)
yay -S rofi-wayland

# Fonts (Hyprland looks great with Nerd Fonts)
yay -S ttf-jetbrains-mono-nerd noto-fonts-emoji
07

Reboot & Launch Hyprland

Reboot your system. SDDM will show your login screen — select Hyprland from the session menu, log in, and your desktop will start.

reboot

# After login, your key bindings:
# SUPER + Return  →  open terminal (kitty)
# SUPER + D       →  open app launcher (wofi)
# SUPER + Q       →  close window
# SUPER + F       →  fullscreen
# SUPER + 1-5     →  switch workspace
# SUPER+SHIFT + E →  exit Hyprland
🎉

You now have a working Hyprland desktop! Check the Hyprland Wiki for advanced configuration — animations, workspaces, window rules, and more.

What to Do After Installing

Your Arch system boots — now what? Here is everything to install and set up so your machine is actually usable, comfortable, and ready for daily life.

⚡

First Thing — Update Everything

Always do a full system update before installing anything else.

# Full update right after first boot
sudo pacman -Syu

# If yay is installed, use this instead (updates AUR too)
yay -Syu
🔤

Fonts

Arch ships with almost no fonts. Install these or your apps and browser will look broken.

noto-fonts
Google Noto fonts — covers almost every language and script
noto-fonts-emoji
Emoji support — without this, emojis show as boxes
ttf-liberation
Free replacements for Arial, Times New Roman, Courier
ttf-jetbrains-mono-nerd AUR
Beautiful coding font with icons — great for terminals
# Install essential fonts
sudo pacman -S noto-fonts noto-fonts-emoji \
               ttf-liberation ttf-dejavu

Terminal & Shell

Make your terminal fast and pleasant to use. Zsh with plugins is highly recommended.

zsh
Better shell than bash — tab completion, themes, plugins
oh-my-zsh AUR
Framework for managing zsh config, themes, and plugins
starship AUR
Fast, pretty terminal prompt that works in any shell
kitty
GPU-accelerated terminal — fast, supports images in terminal
alacritty
Minimal, fast terminal — very popular in the Arch community
zsh-autosuggestions
Suggests commands as you type based on your history
# Install zsh and set it as your default shell

# Install oh-my-zsh (run this as your normal user)
sh -c "$(curl -fsSL https://raw.githubusercontent.com/ohmyzsh/ohmyzsh/master/tools/install.sh)"

# Install starship prompt
yay -S starship
# Add this to the end of ~/.zshrc or ~/.bashrc:

Stay Safe & Secure

Now that Arch is running, follow these practices to keep your system fast and secure.

🔄

Keep System Updated

Run sudo pacman -Syu regularly to get security patches and the latest software.

🔥

Enable Firewall

Install and enable ufw. Run sudo ufw enable to block unwanted connections.

🔑

Use Strong Passwords

Use a unique, long password for your user and root accounts. Consider a password manager like bitwarden.

💾

Backup with Timeshift

Install timeshift to take system snapshots. You can restore your system if something breaks.

📦

Use the AUR Carefully

The AUR has tons of packages. Always read the PKGBUILD before installing. Use yay or paru as your AUR helper.

Read the Arch Wiki

The Arch Wiki is the best Linux documentation in existence. Bookmark it.
